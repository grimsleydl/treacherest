# Issue #22: Fix Role Assignment Logic and Auto-Scaling Control

## Core Problem

For a given game, we need an explicit and clear guarantee that the roles are truly randomly selected from the pool of available roles in each role type, and THEN AND ONLY THEN should each player be given a randomly selected role from the pool of randomly selected roles.

## Required Flow

When starting the game, for each Role Type, randomly select N roles of that role type, where N is the "Count" of roles for that type from the game lobby setup to be used in the game (currently displayed under the .role-type-count trees in the DOM and with the input field being $WhateverRole_count). This will form the "pool" of roles for the game.

Then, randomly assign each player a role from that "pool" of roles, tracking which role is assigned to which player so that we can display e.g. the leader in the UI, track reveals, etc.

When starting the game, we should check that the total count of roles for the lobby meets or exceeds the number of players--for presets this will auto scale based on hard coded values. For custom games if it's less, then we should not allow the game to start.

**What's most important is that we are truly randomly selecting the roles to be used in the game and THEN assigning them (also randomly) to players.**

## Additional Requirements (from Issue #21)

### Auto-Scaling Control Features

Building on the validation system implemented in issue #21, we need to add explicit control over auto-scaling behavior:

#### 1. Auto-Scale Toggle UI
- Add a toggle in the role configuration UI that allows hosts to enable/disable auto-scaling
- Should only be visible for preset configurations (not custom)
- Should integrate with the existing role configuration component
- When enabled, shows auto-scale details and validation messages

#### 2. Auto-Scale Enabled Field
- Add `AutoScaleEnabled bool` field to the `RoleConfiguration` struct
- Default to `true` for preset configurations, `false` for custom
- Used by the role assignment logic to determine whether to auto-scale or use exact counts

#### 3. Integration with Validation System
- The existing `CanAutoScale` method from issue #21 should check this field
- Validation messages should reflect whether auto-scaling is enabled
- SSE updates should include the auto-scale enabled state
