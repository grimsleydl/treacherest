# Issue 14: Thoroughly Reexamine Testing Strategy

## Problem Statement

While implementing the leaderless games feature (issue #9), we discovered that our test suite did not catch the changes needed for the new card image URL scheme (issue #13). This indicates gaps in our testing coverage and strategy that need to be addressed.

## Current Issues with Testing

1. **Missing Integration Tests**: The card image URL changes should have been caught by integration tests that verify the full download → load → display pipeline.

2. **Incomplete Test Coverage**: 
   - No tests for the download scripts (`download_cards.go`, `download_sample.go`)
   - No tests verifying that embedded images load correctly
   - No tests for URL construction and encoding

3. **Build-Time vs Runtime Testing**:
   - Card images are embedded at build time, but we don't test this process
   - No tests verify that the embedded resources are accessible at runtime

4. **External Dependency Testing**:
   - No tests that mock or verify external API calls (mtgtreachery.net)
   - No tests for handling network failures or changed URL schemes

5. **Test Failures Not Blocking**:
   - Some existing tests are failing (e.g., `room_role_config_test.go`) but development continues
   - No CI/CD pipeline to catch these issues early

## Proposed Solutions

### 1. Add Download Script Tests
Create tests for the download scripts that:
- Mock HTTP responses to test URL construction
- Verify correct file naming and storage
- Test error handling for network failures
- Test the URL encoding logic

### 2. Add Integration Tests
Create end-to-end tests that:
- Download a sample card image
- Load it via CardService
- Verify it can be displayed in templates
- Test the full pipeline from URL to display

### 3. Add Build-Time Verification
- Add a test that runs during build to verify embedded resources
- Check that all expected card images are present
- Verify base64 encoding works correctly

### 4. Fix Existing Test Failures
- Fix the failing tests in `room_role_config_test.go`
- Ensure all tests pass before adding new features
- Add pre-commit hooks to run tests

### 5. Add URL Scheme Tests
- Test that URL construction handles special characters
- Test the ID prefix format (e.g., "022 - Traitor - The Gatekeeper.jpg")
- Add tests for URL encoding edge cases

### 6. Improve Test Organization
- Separate unit tests from integration tests
- Add test categories (fast/slow, unit/integration/e2e)
- Create test utilities for common operations

### 7. Add Testing Documentation
- Document how to run different test suites
- Add testing guidelines for contributors
- Create examples of good test patterns

## Implementation Priority

1. **Immediate**: Fix failing tests in `room_role_config_test.go`
2. **High**: Add tests for card URL construction and download logic
3. **High**: Add integration tests for the card loading pipeline
4. **Medium**: Add build-time verification tests
5. **Medium**: Set up pre-commit hooks
6. **Low**: Improve test organization and documentation

## Success Criteria

- All tests pass consistently
- New features cannot be merged with failing tests
- Card URL changes are automatically tested
- Download failures are caught before deployment
- Test coverage increases to >80% for critical paths
